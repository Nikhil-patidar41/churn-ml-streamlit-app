## Problem Statement
Customer churn is a major business challenge in the telecom industry.
This project predicts whether a customer is likely to churn and helps
business teams take proactive retention actions.

## Solution Overview
- Built an end-to-end machine learning pipeline
- Handled class imbalance using SMOTE
- Optimized model using ROC-AUC
- Applied business-driven threshold tuning
- Deployed model using Streamlit

## Model Performance
- ROC-AUC: ~0.84
- Churn Recall: ~80%
- Final threshold selected based on business cost

## Tech Stack
- Python, Pandas, NumPy
- Scikit-learn, Imbalanced-learn
- Streamlit
- GitHub
